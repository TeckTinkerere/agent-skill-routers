# routr-frontend boundaries

| Situation | Use instead |
|-----------|-------------|
| Animate existing components only | `routr-motion` |
| Expo / React Native screen | `routr-mobile` |
| SEO copy, meta tags | `routr-marketing` |
| Library API integration | `routr-integrate` |
| Bug in click handler | `routr-debug` |

## vs routr-motion

- **routr-frontend**: structure, typography, layout, new UI
- **routr-motion**: polish motion on existing UI

Build page first, motion pass second.
