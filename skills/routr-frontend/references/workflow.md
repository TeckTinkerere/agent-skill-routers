# Frontend workflow

## Design brief template

```markdown
## Design brief
**Subject:** [concrete product/feature]
**Audience:** [who uses this]
**Job:** [one sentence — what this page must accomplish]

### Tokens
- Colors: [name #hex] × 4–6
- Display font: [family] — used for [headlines only / etc.]
- Body font: [family]
- Layout concept: [one sentence + optional ASCII wireframe]

### Signature element
[The one memorable thing this page will be known for]

### Self-critique
- [ ] Does NOT read as generic AI template (cream/terracotta, acid-green-on-black, newspaper columns without subject fit)
- [ ] Every choice tied to this brief's subject
```

## Process

1. **Brainstorm** — 2 layout directions in thinking; pick one
2. **Plan** — fill brief template above
3. **Critique** — if interchangeable with any SaaS landing, revise tokens/signature
4. **Build** — derive every color and type from plan
5. **Critique again** — screenshot if possible; cut one decorative element

## Accessibility (from web-design-guidelines)

- Visible focus rings
- Contrast 4.5:1 body text minimum
- Touch targets 44px mobile
- `prefers-reduced-motion`: no essential info in animation only
