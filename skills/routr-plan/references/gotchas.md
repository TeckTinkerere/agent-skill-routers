# Planning gotchas

1. **Solution-first**: User asks for React; problem may need no UI — brainstorm intent first.
2. **Vague success**: "Make it better" — force one measurable metric.
3. **Scope creep in PRD**: Non-goals section is mandatory for large features.
4. **Alias confusion**: Use `grill-me` only — not `grilling`.
5. **Skip explore**: Big change without `routr-explore` on affected modules → rework later.
