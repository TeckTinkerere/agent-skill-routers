# Planning fallback

Use when `brainstorming` and `routr-depth-plan` unavailable.

## Minimum brainstorm

1. Restate user goal in one paragraph
2. List 2 approaches with pros/cons
3. State assumptions explicitly
4. List 3 risks
5. Ask user to confirm approach OR answer open questions

## Minimum PRD gate

Before any implementation:

- [ ] Success metric defined
- [ ] At least 3 acceptance criteria (testable)
- [ ] Non-goals listed
